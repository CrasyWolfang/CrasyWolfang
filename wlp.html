General info

Thank you for trying Wolf's Language Program. This software has been in development since 2016 through a lot of trial and error and has now reached its 11th installation called "Fox Gates". The name partially derives Fushimi Inari Taisha in Kyoto, Japan in which the path is lined with vermilion gates all the way to the top shrine. The name treats the passing of each gate as a step in language learning and in that metaphor, how learning is done in a lot of small steps to the climb towards proficiency. The name also derives from the canine which has a similar appearance to a wolf and is known in Nordic entomology to be a sly and clever animal.

The program runs on Java and requires Java SE Development Kit 17.0.9 to run (see installation) and collects data on user input (see data collection). To use Wolf's Language Program you must agree to the terms of usage which will be assumed if you run the program. You will never be charged any money to use the program on the basis that your data will be collected for research purposes (see data collection).

This program is developed by Crasy Wolfang (contact: forwardtowolf@gmail.com). Feel free to send any suggestions on improvements. Note that this is a single person side project, therefore updates may take a while.

Aims

Languages The goal of the program is to provide a platform for a wide variation of languages without neglecting characteristics of individual languages. The data for languages and entries is to be formatted in a way that makes sense linguistically and the methods for learning to have a scientifically proven base. Therefore the data generated by the program will also be used to improve it.
Digitization The program also has a goal to find a good way to digitize conventional media (textbook and dictionaries) in a structural manner, yet preserve as much data from the original media as possible. This will provide the ability to search for data and to create repeatable practices. The program will also try to provide a variation of methods in which to memorize and learn through digital means based on the resources available.
Performance Small changes will also be made to the program continuously to find methods to increase performance for it to run on older systems, and to make it more user friendly. The user should also be able to run the program offline without compromise in functionality.
As a tool for learners The program should provide a useful tool for learners of supported languages to repeat and practice. The user should be able to use the program to prepare for exams and to practice new material. Note that the user is still expected to own a copy of the media that the program resources are based on and therefore not use this program as a substitute. The program should also provide a way for the learner to compose their own glossary lists and practices which they could share with other students.
As a tool for teachers The program also has a goal to provide a tool for teachers in which they could check material that the student is expected to know, and to propose tasks for in-class based on that material. The teacher should also be able to create practices and handouts and share with their students.
Research Finally the program also has the goal to provide data for research (see data collection). This is used to understand tendencies in learners and to find improvements on how languages are taught. The aspiration is that this data can be used for research papers, but also that it can be published on its own. Furthermore the program should be able to handle the available resources as text corpora.

Terms of usage and data collection

By using Wolf's Language Program you automatically agree to the terms of service and that your data may be collected for (1) research purposes and (2) some data to improve the program.

(1) The program collects data via the user's input and some from feedback by the program. The data collected includes practice results, time information and the user input. This data may also be combined with the specified user target- and text languages. The user will always be made anonymous before their data is shared or results published.

(2) For performance improvements the program might temporarily measure how long it takes to perform certain methods like saving data, searching, etc. The data on program performance will never be published and is purely there to check and improve the program's performance.

The user may at any time request to get information about data collected on them or to request that their data is deleted and they can withdraw from the program. Send an email to forwardtowolf@gmail for requests.

Participation

As a user of Wolf's Language Program your feedback is valuable! Participation in feedback is voluntary but highly appreciated. Via your feedback you can shape the program and make it better. Suggestions and corrections can be small (ex. suggestion to move a button) or large (ex. implement a new feature). These suggestion may be prioritized by demand.

If you would like to, please also send in information if you find a bug prevents the program from running with information on how to reproduce it. You can notice a bug that prevents the program from proceeded if a button is pressed but color doesn't switch back until you move the mouse away from the button. If the color doesn't change at all when you move the mouse away from the button the program is loading. If it gets stuck like this forever there is also an error.

If you want a next text language added to the list of languages please provide information on:

    4. parenthesis (())
    5. square brackets ([])

Please also send a mail if there is anything unclear (the function of a button is unclear or the instructions are unclear).

Also note to that if you edit official resources that came with the program they will not be saved for the official release. Developer notes

The current changes are already known and will implemented in a future release:

  • Problems with resizing

    The size of the window isn't calculated correctly which will cause there to either be a surplus of blank space or so that components in the window are squeezed together. Gently drag the corner of the window and resize it to temporary solve it.

  • Practice setup

    The practice setup is currently more complicated than the average user needs it to be. The advanced options will be made hidden but accessable.

    There are also more task option in the list than is available for the selected material. This is mostly a problem for Japanese (as it has a large variety of data and entry types available). This will be changed to be adapted after the selected material.

  • Practice session

    Audio can at the moment not be detected to be the same. Therefore they do not combine like other material.

  • Resource menu

    Presentation of resources will be improved. This includes folders within the resource, tabs, etc.

  • Resource properties

    Setting up resource data support is an absolute mess and will be revised. Fortunately this is just something you have to deal with if you set up a resource which may not be too often.

  • Profile menu

    The profile menu doesn't show much more than previous practices so far. A better summary of history and other features will be added.

  • Audio

    Playback for audio will be improved and the synchronization menu will get some upgrades.

  • Images

    Image support will be added. Like with audio, each image will be split and synchronized with the resource.

  • Text in books

    Text for reading comprehension will be implemented and added to resources.

  • tone data class

    A class which handles languages with tones (ex. Chinese) will be added.

  • character data class

    A class that handles single characters (ex. kanji, hanji) will be added.

  • Practices

    Other types of tasks are planned for the future (ex. multiple selection).

  • Symbols

    Some fonts do not support symbols. This will be fixed so that symbols work independent from the font.

  • Color themes

    Color themes by user preferences will be added in the future.

Installation

  1. Download and install Java SE Development Kit 17.0.9
  2. Download Wolf Language Program ("Fox Gates") and desired resources
  3. Install resources
  4. Launch program and setup your profile

Download Java

https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html

To run the program you need Java SE Development Kit 17.0.9 which can be downloaded from Oracle's website. Note that due compilation issues only version 17.0.9 has been confirmed to work so far. With any other version the program fails to launch. This is an issue that will be resolved later. Please make sure to have the correct version of Java for the program to launch, and also to download the correct version for your system.

Once downloaded you can open the file and follow the instructions.

Download Wolf's Language Program

https://drive.google.com/drive/folders/0B7SK612su6mzaU5TYm5pV20xVEE?resourcekey=0-lGuNgxAw0niR4WXxIUE7zw

Wolf's Language Program is currently in its test phase (alpha) and can therefore only be found on an unlisted Google Drive folder. Download "Wolf's Language Program.zip" by either selecting the file with the right mouse button or pressing the three vertical dots in the upper right corner and selecting "download." This is also a good time to download the .zip for your target language(s).

Then unzip/extract the files onto anywhere on the computer. This can be done in windows by dragging and dropping the zip-file with the right mouse button and selecting "extract...", or via a compression program like WinRar, 7Zip, etc.

This extracted folder should contain "foxgates.jar" and a folder called "resources." The "foxgates.jar" contains the full program. For easy access, create a shortcut to "foxgates.jar" from your desktop or similar. The "resources" folder is where you install content for the program. Note that the program will generate more files in this folder the first time your start it.

Install resources

https://drive.google.com/drive/folders/0B7SK612su6mzaU5TYm5pV20xVEE?resourcekey=0-lGuNgxAw0niR4WXxIUE7zw

To install resources, first download the .zip of your target language(s) from the google drive folder the same way as you downloaded "Wolf's Language Program.zip". Unzip this/these files anywhere on the computer and move the containing folders into the "resources" folder under the same folder as the language program.

Launch the program

Once you got Java and have installed the program it is ready to be launched for the first time. To launch the program, open "foxgates.jar" with Java(TM) Platform SE binary. This can be set up the first time by pressing on "foxgates.jar" with the right mouse button and selecting "Open with...", selecting "Always use this app to open .jar files" and selecting Java(TM) Platform SE binary in the list. If is not available in the list you can find it by selecting More apps and scrolling down and selecting Look for another app on this PC. From there go to where the folder you installed JAVA in ("C:\Program Files\Java\jdk-17" by default), "bin" and double click on "javaw.exe". Due to some bug in reading/writing you might need to close the program and reopen it again by double clicking on "foxgates.jar", otherwise your profile might not be saved. If your folder only contains the two items (foxgates.jar and resources) you need to restart it.

Menu

Login

When you start the program you will start at the login screen. Here you have two options:

1. login to existing profile logs in the user on existing profile
2. create new profile creates a new profile with the name in the field

To create a new profile, write your desired profile name into the field and press create. Your profile name may not include any slash, colon, star, question-mark, quote mark, lesser/greater than or pipe symbol. When you create your profile the profile settings will automatically open. Read user settings on how to setup your profile.

Main Menu

1. title Contains information about the current version of the program and what profile is logged in.
2. search all Allows you to search through all the material in available resources.
3. practice Opens the practice setup screen.
4. resources Allows the user to browser available material as well as edit and add their own resources.
5. profile Shows you data on your profile.
6. settings Contains preferences and settings for your profile.
7. change profile Logs out current user and takes you to the login-in screen.

Search

The search menu gives the user the ability to search through all of their active resources of chosen language. Note that the search will only look for dictionary entries.

1. language selection selects the target language of search
2. search input a. input field Write your search into this field. The search may contain multiple words of both the target language and translation, etc. Note that the program will only show results of entries that have all of the specified words in it.
b. clear search Clears all search input.
c. search Either press this button or hit enter on the search field to search.
3. search results shows all of the search results

Furthermore, each search result will have a few options:

1. target data main data for the entry
2. text data data for languages specific for the users known languages
3. edit button This button can be used to edit the entry. When pressed it will open up a batch with all of the entries in the search. Read in editing dictionary entries on how to edit.
4. glossary shortcuts If you select the different rows of data you get the option to add them to your resources as glossary via your shortcuts. The glossary you add will only include the selected data.

Dictionary entries are edited in batches. This is so that multiple entries can be added at the same time and so that it is possible for the editor to go back and proof read. It also makes it possible to go through chapters of entries to add or revise data. The user can also select important data for the batch by selecting them on the left (pressing on the title of the data). This will cause the focus to jump between the specified fields when enter is pressed. Also see data classes on how to correctly format data.

1. save button applies and saves changes
2. batch navigation a. go to previous applies changes and goes to the previous entry
b. index in batch index of current entry in batch
c. go to next applies changes and goes to the next entry
3. entry ID* unique ID of entry in resource
4. entry type selector sets the entry type for the entry
5. target data The target data of the entry is specified here. Know that the labels on the left can be selected to jump between relevant data.
6. text data a. text language label specified language for set of text data
b. remove button removes the set of text data
c. text data The set of text data is specified here. Know that the labels on the left can be selected to jump between relevant data.
d. text data language selector sets added text data to language
e. text data add button adds another set of text data to the entry

Data

Dictionary entry

The dictionary entry contains one set of target data and supports multiple instances of text data for multiple languages. It also supports examples in the form of glossary which can be extracted for practices. Dictionary entries can be search for and can be used to create glossary. Dictionary entries are meant to list detailed information mainly for browsing but can also be used for practices.

Glossary

Glossary contains one set of target data and one set of text data for each text language of that resource. Glossary is meant for lists of vocabulary with not too much information and should mainly be used for practices. Glossary cannot be searched for.

Media

The program supports media in the form of audio files. The files must be in .wav format and need to be synchronized into their resource. Media files are stored in .../resources/[resource ID]/media.

Data classes

String

String (text) values are defined as a row of text. These values should be formatted so that they can be used in practices. The following characters have a special function:

character function example
semicolon (;) inserted between multiple values hard; difficult
parenthesis (()) marks words that are optional to raise (above)
square brackets ([]) Marks annotations that are ignored in practices. Also marks the end and start of alternatives. handsome [used for men]
slash (/) marks variation/alternatives (note that it should be used within square brackets) to be [difficult/hard] to understand

Integer

Integer (number) values consist of a full number (no decimals).

Combo

The combo class is a text value with a limited options such grouping or classification.

Audio

The audio class refers to a single audio snippet of a resource. When the audio is available it will be displayed as a button that can be pressed to play the audio. Most tasks also allow the user to press space or 0 (zero) to play the audio.

Tasks

Flash card

The flash card displays one type of data and the user inputs the requested data in the input. Feedback is then shown on the answer and the correct answer. The task card supports strings (text values) and integers (number values) for target data.

Setup

1. passive data selection The data that is displayed for the user.
2. active data selection The target data that the user inputs.

Task

1. target Shows instruction of the expected value.
2. passive data The information that the user will base their answer on.
3. input field The answer is written here. To answer press enter. Note also that when in focus and nothing is written the user can press spacebar or 0 (zero) to repeat audio. Also, the input doesn't care for lowercase/uppercase letters and phonetically identical characters like katakana/hiragana.
4. feedback Displays if the input was correct with "hit" or incorrect with "miss".
5. correct answer Displays the correct answer. Note that the answer can be pressed to show more information.

History

The history displayed the amount of incorrect answers. If the number is pressed the input history will also be displayed.

Paper card

The paper card displays one type of data and the user inputs the requested data in the input. The input is written down on a piece of paper and the user specifies if they think they passed or not.

Setup

1. passive data selection The data that is displayed for the user.
2. active data selection The target data that the user inputs.

Task

1. target Shows instruction of the expected value.
2. passive data The information that the user will base their answer on.
3. input field The answer on written paper and the user then presses enter here to show the correct answer. Note also the user can press spacebar or 0 (zero) when the answer isn't shown to repeat audio.
4. answer input a. miss Press this if your input is incorrect. You can also press space or 0 (zero) to mark the answer as incorrect.
b. hit Press this if your input is correct. You can also press enter to mark the answer as correct.
5. correct answer Displays the correct answer. Note that the answer can be pressed to show more information.

History

The history displayed the amount of incorrect answers.

2. create new shortcut a. input field name for new shortcut of current folder location

Resources

The following resources are provided with Wolf's Language Program.

Chinese

  • Chinese (Lund University)

    Contains the material used for Chinese Studies at Lund University. Has all vocabulary until chapter 10 (beginner's course) together with audio recordings. Also contains extra glossary like adjectives, counters, etc. and material on extra reading: "The Lady in the Painting." The text language will be changed into the original Swedish (extra material currently in English and chapter vocabulary in both Swedish and English) and "The Lady in the Painting" will be revised with correct Swedish translations and audio.

Japanese

  • Genki

    By Banno et al., 1999 (The Japan Times). An Elementary course in Elementary Japanese. Has Genki 1 and Genki 2 second edition. Contains all vocabulary by chapters, and the writing section including both kanji and kana. Chapter names will get a revise and translation into English and audio will be added.

  • Tobira

    Tobira: Gateway to Advanced Japanese Learning Through Content and Media, by Oka et al, 2009 (Kurosio). Contains all vocabulary in the chapters of the main book and all kanji in the reading and writing book. Sentence patterns from the book will later be added into the resource.

  • Kanji Learner's Course

    Kanji dictionary of the Joyo kanji list by Conning, 2013 (Kodansha). The data has been scraped from an external source and will be revised on a later date.

  • A dictionary of Japanese Grammar

    Masaki & Tsutsui's Japanese Basic/Intermediate/Advanced grammar dictionaries (The Japan Times) released in 1986, 1995 and 2008. Currently only contains the basic grammar but will be filled with the other two parts once the Japanese grammar class is resolved.

  • Japanese-Multilingual Dictionary

    The famous JMDict found on integrated browser translators, smart-phone applications and on Jisho.org. Contains the base data from the original dictionary and only English translations. Will be worked on eventually to include more information and other languages.
